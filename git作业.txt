1. Git addd 和git stage 的区别是什么Git add和git stage用法一样，都是把修改的文件存放到寄存区2. Git rm -cached和git rm -f区别是什么-cached是删除存储在暂存区的文件，后者是完全删除文件3. git和svn的区别是什么a. Git,svn都可以用来存储代码，但是git更便于分支管理和合并管理b. 从git 载下来可以查看所有的log, svn 无法查看log4. 筛选出 2018.10.1 到 2018.10.20之间的日志,并且输出为地理图,并且没有做过合并git log --graph --since='2018.10.1' --after='2018.12.20'5. git init和git clone的区别Git init: 初始化本地的仓库，会生成.git文件Git clone:克隆远程仓库到当前文件夹，会一起克隆远程的文件夹6. 每次提交都忽略.idea文件夹里面的东西怎么办在.gitignore添加.idea文件夹7. 如果编辑一个文件之后并且加入了暂存区,但是你后悔了,想把文件恢复到没有修改之前的样子,怎么办首先删除暂存区文件git rm -cached, 然后新建分支git checkout8. 如何检出标签?git tag ‘name’ + 提交编号头部 创建标签，通过git show 'name' 就可以查询到提交的信息
9. git fetch 和 git pull的区别git fetch：是将远程主机的最新内信息拉到本地，并不会更新本地文件。git pull ：则是将远程主机的最新内容拉下来后直接合并
10. 如何添加远程仓库在git网站新建xx仓库并复制html地址，使用git remote add '仓库名‘　+ 复制的地址，就可以用git push/git pull管理远程仓库
